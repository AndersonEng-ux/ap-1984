Anderson Design System · v1.4.0

One governed language for everything Anderson Psomas ships.

Anderson Psomas works for buyers whose own standing rides on the work holding up — under a permit review, a state regulator, a lender's underwriting, a corporate EHS audit. Oil-and-gas and industrial project managers, construction-company owners, county engineers, the lender behind a land deal. None of them move on a pitch; all of them are handing the deliverable to someone stricter than themselves. This system makes every surface read like work that survives that scrutiny — the credibility site, the PPC page, the proposal, Phleet, the quoting tool. Severe, verifiable, unhurried. The restraint is the credibility argument, not a house style.

One token canon, retuned across three registers — editorial, app, and print. Monochrome-plus-one. Zero-radius. A rebrand is a single token edit; every surface follows.

The hard laws

Six laws, ratified by eye and enforced by build. Break one and the lint or the contrast gate fails.

01
Radius is always 0

The geometric lock — --ap-radius: 0 is the only radius token; base.css force-zeros every element, even radios.

02
Type is NHG Medium 500, tracking 0

Neue Haas Grotesk Display/Text at 500 (never 600), 400 for deks. IBM Plex Mono is DATA only — fees, dates, IDs, coordinates. Never prose.

03
Monochrome-plus-one

Ink #111111 + ONE permit red #B32A2E (PANTONE 484). One red action per screen; red links always underlined.

04
Navy is structure, never a voice

Pacific Blue #0C3350 lives in the lockup, rules, and shelves — intensity scales with formality (the navy dial). No navy text token exists.

05
Charts are ink-first

Graphite outlines, ONE red marker (the figure that matters), a navy dashed reference line. Never a rainbow.

06
Three registers retune one canon

Editorial (warm-white #FCFCFA, 16px) · app (white workbench, 15px, motion ≈ 0) · print (816px×1056px, 10px floor, prints white).

How to use

Reach for semantic tokens, never the raw ramp: var(--ap-text-primary) for type, var(--ap-surface-page) for grounds, var(--ap-border-soft) for hairlines. The raw --ap-color-* values exist so the semantic layer can chain to them — components consume the semantic layer and touch raw color zero times.

color: var(--ap-text-primary);          /* ink — body & headings */
color: var(--ap-text-secondary);        /* neutral grey — meta, captions */
background: var(--ap-surface-page);     /* warm-white editorial ground */
border: 1px solid var(--ap-border-soft);/* hairline */
color: var(--ap-text-accent);           /* the one red — key phrases / data emphasis */

--ap-text-muted-large (#8D8D8D, "steel") is the one trap: large display flourishes only, never small/UI/body text — it clears large-text 3:1 but fails small-text AA.

Foundations

Voice & writing

Document-grade. Write like the document the county receives.

Do
  • Numbers, not adjectives — "1,847 Phase I ESAs stamped since 1995".
  • "Fee", never "investment". Name the county. Name the statute.
  • Sentence case everywhere except single-line mono labels.
  • Empty states name the action — "No quotes yet — create the first."
Don't
  • No emoji. No exclamation marks. No "Welcome to" heroes.
  • No value-word grids ("Integrity. Excellence. Innovation.").
  • No animated counters, countdowns, fabricated testimonials.
  • No marketing theater on professional surfaces.

Judgment rules (review-enforced — no lint can catch these)

  • One red action per screen. A primary button exists only where its action belongs to the page; at most one ghost (Back); red links underlined.
  • Mono = data only. Fees, dates, IDs, licenses, coordinates. Mono never sets a sentence; letterspaced caps for single-line labels only.
  • Charts ink-first, one marker. The categorical palette only when hue IS the data — one per real category.
  • Imagery real or absent. Stock never stands in for a named person/place; fallback = ink schematic "NOT TO SCALE"; fiction-safe only.
  • States are mandatory. Every listable view ships empty/loading/error; Toast + Undo for state changes; errors are text, not color.
